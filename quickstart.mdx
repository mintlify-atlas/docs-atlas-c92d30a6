This guide takes you from a fresh clone to logging your first food and transport activity on your local Carbon Tracker instance.

## Prerequisites

Before you start, make sure you have the following:

- **Node.js 18+** — [download from nodejs.org](https://nodejs.org)
- **npm** (bundled with Node.js) or **pnpm**
- **A Supabase account** — the project already has a hosted database; you only need the public URL and anon key from the README

<Note>
  You do not need to create a new Supabase project. Carbon Tracker ships with a pre-configured hosted database. The credentials below are the public anon key — safe to use in a client-side `.env.local` file.
</Note>

## Setup

<Steps>
  <Step title="Clone the repository">
    Clone the project and move into the project directory:

    ```bash terminal
    git clone https://github.com/Gabi2306/tracker.git
    cd tracker
    ```
  </Step>

  <Step title="Install dependencies">
    Install all required packages. Run this command from the project root (the same folder that contains `package.json`):

    <CodeGroup>

    ```bash npm
    npm install
    ```

    ```bash pnpm
    pnpm install
    ```

    </CodeGroup>

    <Warning>
      Run the install command in an **administrator terminal** (Windows) or with the appropriate permissions on macOS/Linux. Without elevated permissions, some native dependencies may fail to install.
    </Warning>
  </Step>

  <Step title="Create the environment file">
    At the project root — the same level as `package.json` — create a file called `.env.local` with the following content:

    ```bash .env.local
    NEXT_PUBLIC_SUPABASE_URL=https://ipztuzsiyryualnqpuwi.supabase.co
    NEXT_PUBLIC_SUPABASE_ANON_KEY=sb_publishable_Xjsy0Yj4mZFkqACUD_SkNQ_ARCEWMYG
    ```

    <Note>
      Both variables are prefixed with `NEXT_PUBLIC_` because Next.js exposes them to the browser bundle. Never add server-side secrets (like a `service_role` key) with the `NEXT_PUBLIC_` prefix.
    </Note>
  </Step>

  <Step title="Start the development server">
    <CodeGroup>

    ```bash npm
    npm run dev
    ```

    ```bash pnpm
    pnpm dev
    ```

    </CodeGroup>

    Next.js will print one or more local URLs to the terminal, for example:

    ```
    ▲ Next.js 16.x
    - Local:   http://localhost:3000
    - Network: http://192.168.x.x:3000
    ```

    Open any of the listed URLs in your browser to reach the app.
  </Step>
</Steps>

## Create an account

<Steps>
  <Step title="Open the registration page">
    The root URL (`/`) shows the login screen. Click **Registrarse** (or navigate directly to `/register`) to open the registration form.
  </Step>

  <Step title="Fill in your details">
    Enter your **name**, **email address**, and a **password**. Click **Registrarse** to submit.

    <Note>
      Supabase Auth sends a confirmation email to the address you provide. Check your inbox and click the confirmation link before signing in for the first time.
    </Note>
  </Step>

  <Step title="Sign in">
    Once confirmed, return to the login page (`/`) and sign in with your email and password. You will be redirected to `/dashboard`.
  </Step>
</Steps>

## Log your first food activity

<Steps>
  <Step title="Open the food log">
    From the dashboard, tap or click **Alimentos**. This navigates to `/log-food`.
  </Step>

  <Step title="Select a food item">
    Use the dropdown to choose a food item — for example, **Res (carne de vaca)** (beef). Over 35 items are available, from high-emission animal products to near-zero-emission plant foods.
  </Step>

  <Step title="Enter the quantity">
    Type the amount in **grams** (e.g., `200`). The form recalculates the estimated CO2 equivalent in real time.
  </Step>

  <Step title="Save the entry">
    Click **Guardar**. You are returned to the dashboard, where today's total now reflects the new entry.
  </Step>
</Steps>

<Tip>
  Start with a realistic portion size — 200 g of beef adds roughly 5 kg CO2e, while 200 g of lentils adds less than 0.1 kg CO2e. This gives you an immediate sense of the relative impact of different foods.
</Tip>

## Log your first transport activity

<Steps>
  <Step title="Open the transport log">
    From the dashboard, tap or click **Transporte**. This navigates to `/log-transport`.
  </Step>

  <Step title="Select a transport mode">
    Choose how you travelled — for example, **Auto** (car), **Bus**, **Moto** (motorcycle), **Taxi**, **Caminando** (walking), or **Bicicleta** (cycling).

    <Note>
      Walking and cycling are zero-emission modes. Selecting them records the trip in your history without adding any CO2e to your daily total.
    </Note>
  </Step>

  <Step title="Enter the distance">
    Type the trip distance in **kilometres** (e.g., `15`).
  </Step>

  <Step title="Save the entry">
    Click **Guardar**. The dashboard updates immediately with the new transport emission.
  </Step>
</Steps>

## Check the dashboard

Navigate to `/dashboard` to review your activity:

- **Ring indicator** — your total CO2e for today, colour-coded by emission level.
- **Comparison to yesterday** — see whether you're above or below yesterday's total.
- **7-day bar chart** — your emissions trend for the current week.
- **Eco-tip** — a personalised environmental tip based on today's emission level.

<Tip>
  Visit `/historial` to see a full chronological list of every logged activity, and `/ranking` to compare your weekly emissions with friends.
</Tip>
